## ‚ú¶ What is This?

The **Adaptive Diagnostic Engine** is an intelligent GRE preparation backend built around **1-Dimensional Item Response Theory (IRT)**. Instead of serving students a fixed set of questions, it continuously estimates a student's ability level and selects the *next best question* ‚ÄĒ always calibrated to be just challenging enough.

After 10 questions, it hands your session data to **Groq's LLaMA 3.3 70B** model to generate a concise, actionable 3-step study plan targeting your exact weak spots.

## ūüßģ Adaptive Algorithm ‚ÄĒ How It Works

The engine uses a **1D IRT-inspired adaptive algorithm** to model and update a student's latent ability in real time.

<br/>

**Starting Point**

Every student begins with a neutral baseline:

```
ability‚āÄ = 0.5     (scale: 0.1 ‚Üí 1.0)
```

<br/>

**Question Selection**

At each step, MongoDB is queried for the question whose difficulty is *closest* to the student's current ability score ‚ÄĒ using an efficient aggregation pipeline:

```
question = argmin |difficulty - ability|
```

<br/>

**Ability Update Formula**

After each response, the ability score shifts toward the truth:

```
‚úÖ Correct:    ability += 0.1 √ó (1 - ability) √ó difficulty
‚ĚĆ Incorrect:  ability -= 0.1 √ó ability √ó (1 - difficulty)
```

This bounded update rule ensures:
- Scores **never exceed** the `[0.1, 1.0]` range
- Progress **converges** naturally ‚ÄĒ large jumps early, fine-tuning later
- Difficulty and correctness **both influence** the update magnitude

<br/>

**Session Completion**

After **10 questions**, the session is marked complete and the `/study-plan` endpoint becomes available.

## ūü§Ė AI-Generated Study Plan

Once a session concludes, the following data is packaged and sent to **Groq's LLaMA 3.3 70B**:

- Final ability score
- Overall accuracy
- Topics where the student answered incorrectly

The model returns a **structured 3-step study plan** that directly targets the student's weak areas ‚ÄĒ not a generic one-size-fits-all guide.

```
Example output:

  Step 1 ‚Üí Focus on [Weak Topic A]: practice 20 questions per day
           with emphasis on elimination strategies.

  Step 2 ‚Üí Review [Weak Topic B] fundamentals using...

  Step 3 ‚Üí Simulate timed conditions by...
```
